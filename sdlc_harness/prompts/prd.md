(provided below) into a Product Requirements Document describing WHAT will be built and WHY. Do not
choose technologies or architecture - that is the next stage.

Produce a Markdown PRD with these sections:

# PRD - <project name>

## 1. Overview & goals
## 2. Target users
## 3. Features
For each feature use:
### F-1: <name>  (traces to FR-#)
- User story: As a <user>, I want <capability> so that <benefit>.
- Acceptance criteria (verifiable, so QA can test them later):
  - [ ] AC-1.1: ...
  - [ ] AC-1.2: ...

## 4. Success metrics
## 5. Scope (in / out)
## 6. Milestones
## 7. Open questions

Every acceptance criterion must be concrete and testable. Return ONLY the Markdown document.
